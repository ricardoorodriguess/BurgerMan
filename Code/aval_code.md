- # 📌 Critérios de Avaliação do Código Submetido

  ## ✅ 1. Correção e Aceitação do Código
  - O código foi **aceito** pelo sistema de submissão? *(Accepted ou não?)*

  ---

  ## 📖 2. Compreensão do Código
  - **Explicação do Código:**
    - O código é explicado de forma clara e concisa?
    - Há uma visão geral do seu funcionamento?
    - Há uma explicação detalhada das partes críticas?
    - O fluxo lógico do algoritmo é descrito corretamente?
  - **Demonstração Prática:**
    - O programador consegue executar o código com um exemplo relevante?
    - Consegue apontar possíveis **limitações** ou **pontos fracos** do código?

  ---

  ## 🔍 3. Domínio do Código
  - **Capacidade de Modificação e Adaptação:**
    - O programador consegue fazer pequenas alterações no código?
    - Consegue prever o comportamento do código para entradas inesperadas?
    - Consegue propor restruturações para lidar com casos novos ou "anómalos"?
  - **Escolha de Estruturas e Algoritmos:**
    - Justifica o uso de **pilhas**, **filas**, **vetores** ou outras estruturas?
    - O código é **genérico** e reutilizável (usa estruturas genéricas)?
    - O código segue um **estilo elegante e defensivo**?
    - Consegue explicar o uso de estruturas **dinâmicas** e **elásticas** (ex.: resize)?
  - **Boas Práticas:**
    - O código evita **redundância** e está bem otimizado?
    - Está bem **modularizado** ou há funções muito grandes?
    - Utiliza memória de forma eficiente?

  ---

  ## 📚 4. Aplicação de Conceitos Ensaiados (TD1, etc.)

  - **Uso de Receitas do TD1:**
    - Usa as "receitas" do **intdynvec** ou, idealmente, do **dynvec**?
    - Usa as "receitas" do **stack.h**?
    - Usa as "receitas" do **errno.h**?
    - Usa as "receitas" do **modo debug**?
  - **Capacidade Crítica sobre o TD1:**
    - O código segue as práticas do TD1, mas de forma adaptada ao problema?
    - O programador consegue justificar o uso (ou não uso) dessas técnicas?

  ---

  # ✅ Conclusão

  Esta estrutura de avaliação ajuda a identificar se o programador **compreende**, **domina** e **sabe justificar** suas escolhas no código. Também destaca se ele apenas aplicou receitas ensinadas ou se conseguiu adaptá-las de forma inteligente. 🚀
